# خارطة الإطلاق التجاري — Bid OS

كل ما تبقّى **عليك أنت** قبل الإطلاق، مرتّبًا حسب الأولوية، مع شرح كل بند: لماذا، من أين، وكيف.
الكود جاهز ومبني (بناء أخضر، 39 اختبار وحدة). ما يلي خدمات خارجية وأعمال تشغيلية وقانونية.

> الرموز: 🔴 حرج (بدونه لا إطلاق) · 🟡 مهم (قبل أول عميل دافع) · 🟢 لاحقًا (تحسين)

## المرحلة 1 — تشغيل تقني (🔴 قبل أي عميل)

### 1.1 🔴 قاعدة البيانات — تم ✅
نفّذت `db:push` + `db:seed` بنجاح. الجداول والباقات وحساب الأدمن جاهزة.

### 1.2 🔴 التخزين — Cloudflare R2
- **لماذا:** كل كراسة ووثيقة تُرفع تُخزَّن هنا. بدونه الرفع يفشل على Vercel (قرصه مؤقّت).
- **من أين:** dash.cloudflare.com → R2 → فعّله (مجاني حتى 10GB، يطلب بطاقة للتحقق فقط).
- **كيف:**
  1. Create bucket → سمّه (مثلًا `bid-os`).
  2. Manage R2 API Tokens → Create API Token → صلاحية **Object Read & Write**.
  3. انسخ: Access Key ID، Secret Access Key، و«S3 endpoint» من R2 → Overview → S3 API.
  4. ضع في Vercel: `STORAGE_DRIVER=s3` · `S3_BUCKET=bid-os` · `S3_REGION=auto` · `S3_ENDPOINT` · `S3_ACCESS_KEY_ID` · `S3_SECRET_ACCESS_KEY`. (`S3_PUBLIC_URL` فارغ — الملفات خاصة.)

### 1.3 🔴 الذكاء الاصطناعي — Anthropic (Claude)
- **لماذا:** هو محرّك تحليل الكراسات كلّه. بدونه التحليل وهمي (mock).
- **من أين:** console.anthropic.com → API Keys → Create Key (يبدأ `sk-ant-`). أضف رصيدًا (Billing).
- **كيف:** في Vercel: `ANTHROPIC_API_KEY=sk-ant-...` و`AI_PROVIDER=anthropic`.
- **التكلفة:** ~0.5$/مستخدم مجاني، و~4–20$ للباقات المدفوعة عند أقصى استخدام (راجع جدول التسعير).

### 1.4 🔴 إكمال متغيّرات Vercel
- راجع `docs/DEPLOY.md` — أضف الـ 18 المتغيّر الناقص (أهمها `AUTH_SECRET`, `STORAGE_DRIVER`, `S3_BUCKET`, `AI_PROVIDER`).
- بعد الإضافة: **Redeploy**.

### 1.5 🔴 اختبار المسار الكامل يدويًا
سجّل حسابًا جديدًا حقيقيًا وجرّب من البداية للنهاية:
تسجيل → استلام OTP → رفع كراسة PDF حقيقية → تحليل → مراجعة المتطلبات/المخاطر/التقييم → توليد عرض Word → سجل القرار.
أصلح أي خطأ يظهر قبل أي عميل. **هذا لم يُختبَر حيًّا بعد.**

## المرحلة 2 — جودة وموثوقية (🟡 قبل أول عميل دافع)

### 2.1 🟡 بوابة الدفع — Moyasar أو Tap
- **لماذا:** بدونها لا تحصيل مالي — المنصّة تجريبية فقط. الطبقة مجرّدة والفواتير ZATCA جاهزة، يبقى الربط.
- **من أين:** moyasar.com أو tap.company → سجّل كتاجر (يتطلب سجل تجاري). احصل على مفاتيح API.
- **كيف:** أخبرني عند جهوزية المفاتيح لأبني التكامل (إنشاء عملية دفع + Webhook لتأكيد + تفعيل الاشتراك).

### 2.2 🟡 رسائل SMS فعلية — Unifonic
- **لماذا:** الآن رمز OTP يظهر في سجلّ الخادم فقط (Logs). للإرسال الفعلي للعملاء تحتاج مزوّدًا.
- **من أين:** unifonic.com → سجّل → احصل على `App SID` واعتمد اسم المرسل (Sender ID).
- **كيف:** في Vercel: `SMS_DRIVER=unifonic` · `UNIFONIC_APP_SID=...` · `SMS_SENDER=اسمك`.

### 2.3 🟡 بريد موثّق — Resend
- **لماذا:** دعوات الأعضاء/الإشعارات. النطاق غير الموثّق تُرفض رسائله.
- **من أين:** resend.com → Domains → أضف نطاقك وأكمل سجلات DNS (SPF/DKIM). ثم API Key.
- **كيف:** `EMAIL_DRIVER=resend` · `RESEND_API_KEY=re_...` · `EMAIL_FROM=Bid OS <no-reply@نطاقك>`.

### 2.4 🟡 عامل المهام — BullMQ + Redis (للكراسات الضخمة)
- **لماذا:** التحليل الآن يجري داخل الطلب (inline). كراسة ضخمة + Claude قد تتجاوز مهلة Vercel (timeout).
- **من أين:** Upstash Redis (مجاني) + خدمة worker على Railway.
- **كيف:** أخبرني لأبني عامل BullMQ منفصل + `QUEUE_DRIVER=redis`. (للبداية بكراسات صغيرة، inline يكفي مع رفع `maxDuration`.)

### 2.5 🟡 مراقبة الأخطاء + النسخ الاحتياطي
- **Sentry** (sentry.io، مجاني) لرصد الأعطال الحيّة. أخبرني لأضيف التكامل.
- **النسخ الاحتياطي:** Supabase يأخذ نسخًا تلقائية على الباقة المدفوعة — فعّلها، أو صدّر يدويًا دوريًا.

## المرحلة 3 — قانوني وتجاري (خارج الكود تمامًا)

### 3.1 🔴 كيان تجاري للمنصّة
- **لماذا:** لتصدر فواتير ضريبية كبائع، وتفتح حساب بوابة دفع.
- **من أين:** سجل تجاري (وزارة التجارة) + تسجيل ضريبي (هيئة الزكاة والضريبة) للحصول على رقم ضريبي.
- ضعه في Vercel: `ZATCA_SELLER_NAME` و`ZATCA_SELLER_VAT_NUMBER`.

### 3.2 🟡 الخصوصية والامتثال (PDPL)
- **لماذا:** تتعامل مع وثائق شركات سرّية → نظام حماية البيانات الشخصية السعودي.
- **المطلوب:** سياسة خصوصية فعلية + اتفاقية معالجة بيانات (DPA) مع العملاء. عندنا صفحة «الشروط والأحكام» فقط — تحتاج مراجعة محامٍ.
- ملاحظة: معالجة الكراسات تتم عبر Anthropic (خارج المملكة) — أفصِح عنها للعميل في DPA.

### 3.3 🟡 الشروط والتسعير
- راجع شروط الاستخدام قانونيًا (المسؤولية، حدود الخدمة).
- ثبّت التسعير النهائي (المقترح: أساسي 149 / احترافي 499 ر.س قبل الضريبة).

## المرحلة 4 — ما قبل النمو (🟢 لاحقًا)

| البند | الفائدة |
|---|---|
| اختبارات E2E (Playwright) | منع الأعطال عند التحديثات |

| تكامل ZATCA Phase 2 | ربط الفوترة المباشر |

## ملخّص الحدّ الأدنى لإطلاق تجريبي (Beta) الأسبوع القادم
1. ✅ قاعدة البيانات (تم).
2. ربط R2 (1.2).
3. مفتاح Claude (1.3) + إكمال متغيّرات Vercel (1.4).
4. اختبار المسار الكامل يدويًا (1.5).

بهذه الأربعة تعمل المنصّة بالكامل لتجربة حقيقية (بلا تحصيل مالي حتى تربط الدفع).
للإطلاق **التجاري الكامل**: أضف بوابة الدفع (2.1) + الكيان التجاري (3.1) + الخصوصية (3.2).
